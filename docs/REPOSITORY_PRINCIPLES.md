# ψ-GLIF Repository Principles

Version: 0.1  
Status: Active  
Scope: Public Repository

---

## 1. Purpose

This document defines the operational principles of the ψ-GLIF public repository.

These principles govern:

- human contributors
- software tools
- archive agents
- automated validators
- external AI systems
- future repository integrations

All repository activity must remain consistent with these principles.

---

## 2. Human Authority

Human approval is the final authority for publication.

No software, agent, model, script, or automated process may independently decide that an artifact is ready for public release.

Automation may:

- inspect
- validate
- compare
- classify
- report
- recommend

Automation may not:

- approve publication
- publish private material
- infer consent
- bypass review
- redefine repository boundaries

The final publication decision always belongs to an authorized human reviewer.

---

## 3. Private by Default

Every artifact is private by default.

An artifact remains private until it has completed the required review process.

The absence of a privacy label does not make an artifact public.

The absence of a restriction does not imply permission.
